# Contributing to SentinelFlow AI

We love feedback and contributions! To contribute to SentinelFlow AI, please follow these guidelines:

## Code Style Guide
- **Backend (Python):** Follow PEP 8 style standards. We use `structlog` for logging. Ensure all new logic is covered by unit tests in `tests/`.
- **Frontend (Next.js):** Write modern React 19 functional components using TypeScript types.
- **Mastra (TypeScript):** Follow TypeScript standards. Keep agents and tools modular.

## Pull Request Process
1. Fork the repository and create your feature branch from `main`.
2. Commit your changes with descriptive messages following conventional commits format.
3. Verify that all tests pass (`pytest` inside the backend directory).
4. Submit a Pull Request targeting the `main` branch.
